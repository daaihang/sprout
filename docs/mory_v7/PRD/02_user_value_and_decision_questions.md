# 02. User Value And Decision Questions

## 1. Core User Questions

v7 should answer five long-term user questions.

| User question | Semantic need | Current gap | v7 capability |
| --- | --- | --- | --- |
| How should I plan a major life decision? | Past decisions, emotions, outcomes, similar contexts | Notes do not connect decisions over time | Decision profiles, context pack, reflection evidence |
| How has my relationship with someone changed? | Person identity, interaction history, relationship changes | Entity graph is too thin and hard to correct | PersonProfile, portrait jobs, relation-change detection |
| What are my emotional patterns? | Mood trend, triggers, tone, ambiguity | Mood is text and one inferred label | AffectSnapshot, tone hints, correction events |
| What value is hidden in small fragments? | Pattern discovery, interests, repeated themes | Search and graph are not fed into analysis | Context retrieval before Analyze |
| How can I capture context without effort? | Location, music, photos, links, system suggestions | Foreground capture is still the main path | App Intents, Share, Journaling Suggestions, background jobs |

## 2. Day-One Value

v7 should avoid waiting weeks before feeling useful.

Early value sources:

- self profile onboarding,
- lightweight person relationship questions,
- daily question with freeform answer,
- context pack over the first 5-12 memories,
- "this may matter later" hints with evidence.

## 3. Long-Term Value

Long-term value comes from:

- stable profiles,
- correction history,
- repeated evidence,
- changing relationship patterns,
- decision status changes,
- emotion trend deltas,
- background review jobs.

## 4. Product Metrics

Track:

- context pack hit rate,
- question answer rate,
- correction rate,
- repeated correction recurrence rate,
- identity merge accept/reject rate,
- profile usefulness rating,
- daily/weekly return from notification,
- reflection save/dismiss ratio.
