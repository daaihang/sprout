# 01. Product Vision

## 1. 产品名称与一句话定义

产品名：`Mory`

一句话定义：

> Mory 是一个私有优先的个人记忆操作系统，用来保存、组织、连接和重新理解生活中的人、片段、阶段与选择。

对外英文表达建议：

> A private memory system for the people, moments, phases, and choices that shape your life.

## 2. 为什么要做这个产品

用户长期丢失的往往不是信息本身，而是意义结构：

- 某个人如何反复出现
- 某种情绪如何逐渐形成
- 某个阶段如何开始和结束
- 某个决定为什么会发生
- 某些生活片段之间为什么会彼此关联

Mory 的目标不是堆更多记录，而是把零散材料变成可以被长期操作的私人记忆系统。

## 3. 产品使命

### 3.1 功能性使命

帮助用户：

- 低负担捕获生活材料
- 长期保存多模态记忆
- 按空间、关系和阶段组织内容
- 在回看与反思时获得真正有用的结构

### 3.2 情感性使命

帮助用户减少两种遗憾：

- 不想忘记重要的人与时刻
- 不想反复掉进相同的模式

### 3.3 结构性使命

定义一种以记忆对象、关系、阶段和反思为中心的个人软件，而不是以文件夹或时间轴为中心的软件。

## 4. 北极星

北极星不是生成多少 AI 文案，而是：

> 用户是否越来越容易保存、找回、连接并重新理解真正重要的生活材料。

建议北极星指标：

`Meaningful Memory Retrieval Rate`

## 5. 核心承诺

1. 内容首先属于用户。
2. 短文本、照片、语音、地点和线索都可以成为正式记忆材料。
3. 空间摆放和相邻关系也是记忆的一部分。
4. AI 负责帮助用户理解，不替用户捏造人生结论。

## 6. 产品边界

Mory 不应被做成：

- 通用 PKM
- 协作工具
- 社交媒体
- 公开社区
- 依赖云端托管全部私密内容的服务

Mory 应坚持：

- 私有优先
- 多模态记忆优先
- 回看与理解优先于生产感
- 长期价值优先于短期热闹

## 7. 产品叙事

对外叙事统一为三层：

1. 记住重要的人与时刻
2. 把生活碎片组织成可回看的私人记忆空间
3. 帮助用户看见关系、模式与人生阶段

## 8. 长期目标

形成一个可持续的 `Personal Memory OS`：

- `Artifact` 承载内容事实
- `Composition` 承载空间组织
- `Entity Graph` 承载长期关系
- `Temporal Arc` 承载阶段结构
- `Reflection` 承载高价值 AI 输出
