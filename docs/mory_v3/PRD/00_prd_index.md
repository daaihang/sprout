> 文档版本：PRD v4.0 draft

## 1. 文档目标

本套 PRD 用于替代旧版“人物 + 决策”叙事下已经与当前项目不完全一致的产品文档。

旧版 PRD 的价值仍然存在：

- 它准确抓住了“重要的人”和“重要的决定”这两个情感锚点
- 它明确了海外 iOS 市场、隐私叙事、独立开发定位
- 它具备较好的竞争分析与商业化直觉

但旧版 PRD 也已经暴露出两个问题：

1. 产品定义仍然偏 `people / decision journal`
2. 没有把空间 UI、AI 语义层、长期记忆结构整合成同一个世界观

因此新版 PRD 目标是：

- 保留旧版情感锚点和市场判断
- 升级产品定义为 `Personal Memory OS`
- 重新定义产品核心对象与功能边界
- 明确 AI 的正确职责
- 给出适合当前工程现实的迭代路径

## 3. 新版 PRD 的统一判断

Mory 不再只是：

- 一个 AI 日记 App
- 一个人物关系记录工具
- 一个决策复盘工具

它更准确的长期定义是：

> 一个以私有数据为基础、以空间组合为界面、以语义记忆为中层、以 AI 反思为高层的个人记忆操作系统。

这个定义会影响：

- 功能优先级
- 首页结构
- 数据模型
- AI 角色
- 未来付费点

## 4. 适用原则

阅读本套 PRD 时请遵守以下原则：

- 先看“产品世界观”，再看单点功能
- 先看“核心对象”，再看页面
- 先看“长期可持续结构”，再看短期功能诱惑
- 一切需求判断都要回到“是否强化统一记忆本体”
