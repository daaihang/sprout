## 2. 与早期 v4 文档的关键修正

### 2.1 自动上下文采集

早期文档写法是“保存时静默采集上下文”。当前实现已改为：

1. `CaptureComposerView` 打开后自动采集 location / weather / music 候选。
2. 成功候选默认勾选。
3. 用户可取消某个候选，也可刷新。
4. 保存时将用户 artifact 与 selected context candidates 一次性写入。
5. 保存后不再追加 late context。

理由：第一次 AI 分析必须使用稳定 artifact 快照，避免保存后追加上下文导致分析输入、UI 展示和派生数据不一致。

### 2.2 AI Pipeline 速度优化

早期文档提出 Analyze 和 Reflection 可同时开始。当前不采用该方案。

当前原则：

1. Analyze 必须先完成。
2. Reflection 使用 Analyze 的 salience、reflectionHint、entities 和 gate。
3. 后续如做并行，只做 Analyze 完成后的安全并行：Graph/Arc 与 Reflection 并行。

理由：Reflection 是核心语义输出，不能为了速度丢失 Analyze 上下文。

### 2.3 Home Board arc 状态

早期文档写 `active arc`。当前代码中可展示故事线使用 `accepted arc`。

后续如要新增 `active`，需要先定义 candidate -> active -> accepted 的状态迁移和 UI 行为。

## 4. 真机验证准备

### 4.1 Xcode / Apple Developer

确认以下 capability 和权限文案存在：

| 能力 | 需要 |
|------|------|
| Sign in with Apple | Apple Developer capability + entitlements |
| WeatherKit | Apple Developer capability + entitlements |
| MusicKit | Apple Developer capability + entitlements |
| Location | `NSLocationWhenInUseUsageDescription` |
| Microphone | `NSMicrophoneUsageDescription` |
| Speech | `NSSpeechRecognitionUsageDescription` |
| Photos | `NSPhotoLibraryUsageDescription` |
| Apple Music | `NSAppleMusicUsageDescription` |

### 4.2 本地后端

真机访问 Mac 本地服务时不要使用 `127.0.0.1`，因为真机的 `127.0.0.1` 指向手机自己。

步骤：

1. Mac 和 iPhone 连接同一 Wi-Fi。
2. 查 Mac 局域网 IP，例如 `ipconfig getifaddr en0`。
3. 启动 Go server，监听 `0.0.0.0:8080` 或确保局域网可访问。
4. 真机 build 必须覆盖 `MORY_API_BASE_URL=http://<Mac局域网IP>:8080`。当前项目默认只有 simulator 使用 `127.0.0.1:8080`，iphoneos 默认指向线上 Fly 地址。
5. 在 iPhone Safari 打开 `http://<Mac局域网IP>:8080/healthz`，必须返回 OK。
6. Go 日志保存到 `/tmp/mory-quality-go.log`。

本地启动示例：

```bash
cd /Users/z14/Documents/sprout/server
go run ./cmd/server 2>&1 | tee /tmp/mory-quality-go.log
```

真机构建示例：

```bash
MAC_IP=$(ipconfig getifaddr en0)
xcodebuild \
  -project /Users/z14/Documents/sprout/mory/mory.xcodeproj \
  -scheme mory \
  -configuration Debug \
  -destination 'generic/platform=iOS' \
  MORY_API_BASE_URL="http://${MAC_IP}:8080" \
  build
```

如果公司网络或公共 Wi-Fi 阻止设备互访，优先使用手机热点或同一私有路由器。

## 5. 真机验证清单

### 5.1 Auth persistence

| 步骤 | 预期 |
|------|------|
| 安装 app，Apple 登录 | 登录成功 |
| 杀掉 app 重开 | 不要求重新登录 |
| 断网重开 | 可保留本地 session 或进入可解释的降级状态 |
| 第二天/多天后重开 | token refresh 正常，不回登录页 |

### 5.2 Capture + Context

| 场景 | 示例输入 | 预期 |
|------|----------|------|
| 文字 + location/weather | “Met Alex after lunch.” | Composer 出现地点/天气候选，默认勾选；保存后 detail 有 context artifacts |
| 取消上下文候选 | 取消 weather，只保留 location | 保存后没有 weather artifact，AI prompt 不包含 weather |
| 刷新候选 | 打开 Composer 后点刷新 | 候选更新时间变化，保存只用最新 selected 候选 |
| 权限拒绝 | 拒绝 Location | 保存不失败，候选显示不可用/启用入口 |

### 5.3 WeatherKit

| 步骤 | 预期 |
|------|------|
| 授权位置后打开 Composer | 3 秒内出现 weather candidate |
| 保存记忆 | MemoryDetail 展示 weather artifact |
| 跑一次 AI 分析 | Go 日志或 debug report 中 artifact context 包含 weather summary |

### 5.4 MusicKit

| 步骤 | 预期 |
|------|------|
| 在 Apple Music 播放一首歌 | 系统正在播放 |
| 打开 Composer 并授权 Music | 出现 music candidate |
| 保存记忆 | MemoryDetail 展示 track / artist |
| 停止播放后再打开 Composer | 不生成 music candidate，但保存不失败 |

### 5.5 Audio

| 场景 | 示例 | 预期 |
|------|------|------|
| 中文短录音 | “今天和 Linh 讨论 launch scope。” | Speech 转写出现，用户可编辑，保存后 textContent 参与分析 |

| 混合语言 | “今天和 Alex talk about launch scope.” | 不崩溃，转写可编辑，AI 不过度发散 |

### 5.6 Photo

| 场景 | 预期 |
|------|------|
| 普通照片 | 生成 thumbnail 和 Vision label summary |

## 6. 真机通过标准

v4 进入 RC 前至少满足：

1. Auth persistence 连续冷启动通过。
2. 文字、照片、录音、链接均可保存并进入 AI 分析。
3. location/weather/music 在授权和未授权状态都行为正确。
4. 删除、编辑、重跑不会留下孤儿 arc/reflection/link/edge。
5. Quality Core Batch 在真机或稳定模拟器环境可重复导出报告。
6. 端到端 AI 分析耗时有真实数据；若仍大于 15s，必须明确是 release blocker 还是可接受 beta 限制。
