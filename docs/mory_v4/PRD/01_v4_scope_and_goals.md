# 01. V4 Scope And Goals

## 1. 版本定位

v4 是 Mory 从 "能跑通" 到 "能用起来" 的版本。

v3 证明了五层本体和端到端链路是可行的。但 v3 的 capture 只有纯文字，用户打开 app 感受到的是一个文字日记工具，而不是一个记忆操作系统。

v4 要让用户能：

1. 拍一张照片，系统自动理解内容并纳入知识图谱
2. 录一段话，系统自动转文字、润色、解析
3. 记录时自动带上天气、地点、正在听的音乐，并允许用户在保存前取消
4. 分享一个链接，系统自动提取标题和摘要
5. 打开 app 时首页有真实的、有意义的卡片

## 2. 版本目标

### 2.1 P0 目标（版本必须交付）

| # | 目标 | 验收标准 | 量化指标 |
|---|------|---------|---------|
| P0-1 | 图片 AI 解析 | 用户选取照片后，系统生成描述并纳入正常解析流程 | 照片 → text artifact 生成延迟 < 5s（本地 Vision）|
| P0-2 | 录音转文字 | 用户录音后，iOS Speech 转文字，服务端润色 | 转写准确率 > 90%（中英文），延迟 < 10s/分钟 |
| P0-3 | 天气自动采集 | Composer 打开后生成天气候选，默认勾选，保存时一次性写入 | 用户授权且 WeatherKit 可用时生成候选 |
| P0-4 | 地点自动采集 | Composer 打开后生成地点候选，默认勾选，保存时一次性写入 | 用户授权后生成候选 |
| P0-5 | 音乐自动采集 | Composer 打开后生成当前播放音乐候选，默认勾选，保存时一次性写入 | MusicKit 授权且有播放时生成候选 |
| P0-6 | AI 速度优化 | 端到端分析时间从 53s 降到可接受范围 | 目标 < 15s |
| P0-7 | 认证持久化 | Apple 登录后 token 持久化，冷启动不需要重新登录 | 连续 7 天冷启动 0 次重新登录 |

### 2.2 P1 目标（版本应当交付）

| # | 目标 | 验收标准 |
|---|------|---------|
| P1-1 | 链接 URL 预览 | 用户粘贴链接后，系统提取 title / description / og:image |
| P1-2 | 主页卡片数据连接 | Today Board 展示真实的最近记忆、故事线、感悟 |
| P1-3 | AI 内容把关 | 实体去重、故事线误判检测、感悟质量过滤 |
| P1-4 | 联系人提及 | 记录时可手动 @ 通讯录或 Mory 内部人物 |

### 2.3 P2 目标（版本可以交付）

| # | 目标 | 验收标准 |
|---|------|---------|
| P2-1 | 票据识别 | 照片中的机票/演唱会门票结构化提取 |
| P2-2 | Share Extension | 从其他 app 分享内容到 Mory |
| P2-3 | iCloud 同步 | CoreData + CloudKit 基础同步 |

## 3. 非目标

以下明确不在 v4 范围内：

- HealthKit 集成（P2-P3，需要产品决策）
- WidgetKit / 灵动岛 / Siri（P3 平台集成）
- 跨天/跨阶段长图导出（P4）
- 付费/订阅系统（独立版本）
- 社交分享功能

## 4. 成功指标

v4 发布后，以下指标应当成立：

| 指标 | 基线（v3） | 目标（v4） |
|------|-----------|-----------|
| 单次 capture 可附加的 Artifact 类型 | 3（text/photo/audio，但 photo/audio 无 AI） | 6（text/photo/audio/weather/location/music，全部有 AI 或自动采集） |
| 端到端分析延迟 | ~53s | < 15s |
| 冷启动需要重新登录 | 每次 | 0 次 |
| 首页有意义的卡片数 | 0（空态或静态） | ≥ 3（最近记忆 + 故事线 + 感悟） |
| 每条记忆的平均上下文维度 | 1（文字） | ≥ 3（文字 + 地点 + 天气） |
